HyperText Markup Language — From Structure to Semantics

01 What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It describes the structure and content of a webpage using elements and tags.


  • HyperText — Text that links to other pages (hyperlinks)
  • Markup — Tags that annotate/structure content
  • Language — A set of rules browsers understand

HTML is NOT a programming language — it has no logic or variables. It is a markup language.

02 Basic HTML Structure

Every HTML page follows this skeleton structure:

<!DOCTYPE html> <!-- Declares HTML5 --> <html lang="en"> <!-- Root element --> <head> <!-- Meta info (not visible) --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Page Title</title> <link rel="stylesheet" href="style.css"> </head> <body> <!-- Visible content --> <h1>Hello World!</h1> <p>This is a paragraph.</p> </body> </html>
▶ <html> ├─ <head> — metadata, title, links ├─ <meta> └─ <title> └─ <body> — all visible content ├─ <header> ├─ <main> ├─ <h1>, <p>, <div>... └─ <footer>
03 Elements & Tags

Opening + Closing Tag

<p>Content here</p>

Most elements have both an opening and closing tag.

Self-Closing Tags

<br> <hr> <img> <input> <meta> <link>

These tags do not need a closing tag.

<!-- Anatomy of an element --> <a href="https://google.com" target="_blank">Visit Google</a> <!-- ↑ tag ↑ attribute ↑ value ↑ content ↑ closing tag -->
04 Headings

HTML has 6 heading levels — <h1> is the largest/most important, <h6> is the smallest.

<h1>Heading 1 — Main Title</h1> <h2>Heading 2 — Section</h2> <h3>Heading 3 — Sub-section</h3> <h4>Heading 4</h4> <h5>Heading 5</h5> <h6>Heading 6 — Smallest</h6>

Use only one <h1> per page for good SEO. Headings should follow a logical order.

11 <div> vs <span>

<div> — Block Container

Used to group block-level elements. Takes full width, starts on a new line. Used for layout sections.

<div class="box"> <p>Grouped content</p> </div>

<span> — Inline Container

Used to style a part of text inline. Does NOT start a new line. Used for inline styling.

<p>There <span style="color:aqua">are three</span> types</p>
12 Semantic HTML5 Tags

Semantic tags describe the meaning of content — better for SEO, accessibility, and readability.

<header> <!-- Top of page / section --> <nav> <!-- Navigation links --> <main> <!-- Primary content (only one per page) --> <section> <!-- Thematic group of content --> <article> <!-- Independent content (blog post, news) --> <aside> <!-- Sidebar / supplementary content --> <footer> <!-- Bottom of page / section --> <figure> <!-- Image + caption container --> <figcaption> <!-- Caption for figure --> <details> <!-- Expandable content --> <summary> <!-- Visible heading for <details> --> <time> <!-- Date/time value --> <address> <!-- Contact info -->

16 Block vs Inline Elements
▣ Block Elements

Take full width. Start on new line.

<div><p><h1-h6><ul><ol><li><table><form><header><section><article><footer><main><nav>
▬ Inline Elements

Take only needed width. No new line.

<span><a><img><strong><em><b><i><u><br><input><button><label><code>
